The problem this solves

A lab's whole lifecycle in one system

Research is hard to manage because it spans five different disciplines — each with different failure modes.

📋
Winning the work
Reading RFPs, brainstorming ideas, writing proposals — with faculty review — takes weeks of coordination.
🔬
Doing the science
Theory, ML, domain knowledge, and experiment design all need specialists working in parallel, not sequence.
🔍
Catching the flaws
Bad baselines, data leakage, unsupported claims. Verification that just reads — rather than runs — misses them.
📊
Reporting honestly
Progress reports, annual reports, slide decks — produced from verified findings, not from memory.
🤝
Staying in control
AI should draft, not decide. Every proposal, edit, and external send needs a human approval, on the record.
✓
One system, all five
The Cambium is one portable Claude project that handles all of this — with discipline, not magic.

The organization

11 councils, 45 workers — all answering to you

Smart-Tier models: Opus only on the critical reasoning path; Sonnet everywhere it's sufficient. Your model for synthesis and final prose.

Orchestration
2 workers · inherit (your model)
Orchestrator dispatches, synthesizes; Document Office is the only writer — gated on your approval.
Pre-Award Office
4 workers · Sonnet + Opus
RFP-Analyst → Ideation-Facilitator → PI → Proposal-Writer. Wins the project.
Faculty
1 parameterized agent · Opus
Summoned per discipline: forestry, stats, math, CS, ML, AI, economics, biometrics… Unlimited fields.
Scouts
3 workers · Sonnet
Prior-art, spatial methods, and tree-ML literature. Web breadth — verify citations, never invent them.
Labs
3 workers · Opus + Sonnet
Theory (Opus), ML, Domain — build the methods and the load-bearing theorems.
Verification
4 workers · Opus + Sonnet
Three adversarial Opus boards run the code. They do not opine — they reproduce. Code-verified beats asserted.
Execution
2 workers · Sonnet
Sim lab and Ablation lab run experiments, measure what each component is worth, flag redundancy.
Reporting Office

Progress / annual reports and slide decks from verified content. You release at G5/G6.
Support Staff

Record-keeper, Librarian, RA, Figures, Outreach, Integrity Officer, Data Steward, Office-Manager + more.
Honesty is structural, not optional

The evidence contract & governance

Every claim carries a tier. Every number must trace to a script. The writer cannot strengthen a claim beyond its proof.

Claim tiers (on every finding)

ProvedTheorem with a checked proof
Code-verifiedScript ran it; cite command + result
AssertedStated in a doc, not run or proved
OpenAcknowledged, unresolved

Rule: when two agents conflict, code-verified beats asserted — the agent who ran the code wins.

Severity (P-level)

P0Fatal — invalid claim, leakage, fabrication. Blocks release.
P1Weakens defensibility — overclaim, missing baseline.
P2Polish — style, minor clarity.

Six human gates — non-negotiable

  • G1 — pursue this RFP?
  • G2 — which idea advances?
  • G3 — finalize & submit proposal?
  • G4 — apply P0 fixes to the work?
  • G5 — release the report?
  • G6 — publish / external send? (Director only)

Standing principles

  • Evidence-or-silence: no claim without file:line or command+output
  • Never fabricate a citation — flag unverifiable refs
  • Single-writer per file; no clobbering peers
  • Read-only during review; only Document Office edits
  • Opus on critical-reasoning path; Sonnet where sufficient
