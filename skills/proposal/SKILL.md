# Proposal — assemble and pressure-test a submission

Thin Cambium-way wrapper. Hand off to the **Orchestrator**; run the proposal phase. Assumes an RFP
brief and a chosen idea already exist (run `rfp-intake` first if not).

## Run it
1. **Pre-Award · principal-investigator** — lock the Specific Aims and the central argument for why
   this wins on the solicitation's own criteria.
2. **Pre-Award · proposal-writer** — draft Significance, Innovation, Approach (aims → methods →
   expected outcomes → pitfalls & alternatives → timeline), mapped section-by-section to the review
   criteria and weights.
3. **Pre-Award · budget-officer + grants-compliance** — budget + justification to the rules, and the
   required compliance documents (biosketches, current & pending, facilities, data-management plan).
   For a deep budget pass, use the `budget` skill.
4. **Verification · referee** + **Governance · research-conduct-officer** — score the draft against
   the venue rubric and check authorship/COI/integrity before anything goes out.

Keep a findings ledger; every claim carries an evidence tier (proved / code-verified / asserted / open).

## Gate
Stop at gate card **G3 — "finalize & submit?"**: decision · options · risks · recommendation ·
APPROVE / REVISE / REJECT. **Never submit** without the Director's approval — the human sends it.
