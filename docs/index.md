# Cambium documentation

Cambium is a research-institute-in-a-repo: an orchestrator plus councils of named
specialist agents, a live run board, an evidence ledger, and human approval gates.
This site is the searchable, versioned home for the deeper docs. The full project
overview lives in the repository README on GitHub.

## Where to go next

- **Start here** covers installation, your first run, day-to-day use, and the FAQ.
- **Concepts** explains the institute model: the lifecycle, roles, faculty roster,
  the presentation contract, the model router, and the skills map.
- **Governance and policy** holds the AI policy, AI governance, the AI-use
  statement, research conduct, the output contract, and the tool policy.
- **Reference** is the operator and developer material: MCP integration, evals,
  efficiency, the multi-institution architecture, autorun, the development
  playbook, the roadmap, and the decision log.

Use the search box at the top of any page to jump straight to a term.

## The short version

- The **orchestrator** decomposes a goal, dispatches the real named specialists and
  faculty, merges their work into one ranked decision, and enforces the gates.
- **Councils** (Pre-Award, Labs, Execution, Verification, Reporting, Governance,
  and more) do the actual work, each with its own agents.
- Nothing substantial finalizes without a **human approval gate**. Verified results
  only; claims never run ahead of their evidence tier.
